# Documentation Init Contract

Use this contract before and after generating initial docs.

## Before Editing

```markdown
# Documentation Init Proposal

## Target Surface

docs repo / code repo / single repo / temporary report

## Docs Root

Examples: `.openplan/`, `docs/`, repo root, absolute path, separate docs repo.

## Human Goal Or Intent

## Existing Authority

## Proposed Scaffold

## Files Not Generated

## Assumptions And Pending Confirmation

## Why This Is The Minimal Useful Structure

## Human Decisions Required
```

## After Editing

```markdown
# Documentation Init Report

## Generated Paths

Paths are relative to:

## Source Evidence

## Assumptions Marked In Files

## Follow-Up Skills

- openplan-garden:
- openplan-record:
- openplan-handoff:

## Residual Risk
```

## Durable Intent Rule

Generated text may describe:

- explicit human statements;
- evidence-backed project facts;
- clearly labeled assumptions;
- pending questions.

Generated text must not convert an agent's guess into durable human intent.
